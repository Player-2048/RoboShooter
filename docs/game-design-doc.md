## 一、游戏概述

**游戏类型：** 搜打撤（Search-Fight-Extract）第一人称射击游戏

**核心设定：** 玩家扮演远程操控者，在安全地点通过数据传输链路遥控**机器人载体**进入危险区域执行任务。可控核聚变已普及，能源不再稀缺；真正的稀缺资源是**材料和加工能力**。

**与同类游戏的关键差异：**

| 传统搜打撤 | RoboShooter |
|-----------|-------------|
| 人类干员，死了就是死了 | 机器人载体，损毁=装备丢失，操作者安全 |
| 单一血条 | 双血条：机械耐久 + 信号强度 |
| 局外整理装备 | 局内随时管理仓库、交易、加工 |
| 固定地图认知 | 动态地图建模，永久解锁，下局保留 |
| 个人仓库独立 | 团队共享背包空间，协作收益 |
| 无算力概念 | 算力池共享，黑客攻防，队友可操控你的载体 |

## 二、核心对局循环

### 2.1 战前阶段

1. **选择地图** — 矿场 / 维和地图
2. **匹配队友** — 3人或4人（可能同时开放两种模式）
3. **准备空间（更衣间）** — 与队友商谈阵容：每个人选什么机器人、什么装备配置
4. **部署进入** — 进入目标地图

### 2.2 局内阶段

- 搜刮物资、战斗、完成任务
- 可随时访问**内置论坛**查阅攻略/情报
- 可随时访问**商店**预购下局装备
- 可随时进行**装备维修、物品合成、材料熔炼、拆解**
- 可**预出售**物资（撤离前锁定交易）

### 2.3 撤离

- **多个撤离点** + 多类型撤离点
- **强制倒计时**：地图存在基于通讯频率的信号干扰，随时间推移逐渐增强
- 倒计时结束时玩家**彻底断联**——所有物资和装备全部丢失
- 撤离不需要全队一起：可牺牲队友拖住敌人，主力带关键物资走
- 撤离后收益**事后分配**

### 2.4 机器人损毁

- 机器人本质上是玩家带进对局的装备，损毁 = **永久丢失**
- 经济崩盘时提供**最基础廉价机器人**：
  - 性能极低
  - 用该机器人带出的物资会被强制扣除一部分作为购买成本
  - 可理解为"额度无限、先用后付"

## 三、经济与成长

### 3.1 核心货币

- **资源本位制**：货币绑定矿石和制造蓝图
- 可在游戏内银行/机构用恒定价格兑换材料
- 无删档机制

### 3.2 加工能力变现

- 不同玩家发展程度不同 → 加工能力差异
- 低加工能力玩家想要高端装备 → 付费使用高端加工设备
- 费用以绑定资源的货币支付

### 3.3 获取装备路径

| 方式 | 说明 |
|------|------|
| 购买原材料 | 在局内/局外商店购买 |
| 局内获取原材料 | 搜刮带出 |
| 加工零件 | 原材料 → 零件（需要蓝图） |
| 购买零件 | 直接购买，跳过加工 |
| 组装成品 | 零件 → 机器人/武器 |
| 蓝图解锁 | 解锁对应零件的制造方法（矿石→金属锭→特定流程→成品） |

### 3.4 交易与安全

- 加工器械和场地可出售（方便局外交易）
- **二级仓库**：防止借号后被恶意毁号
- 核心资产放入二级仓库需额外验证

### 3.5 单人 vs 组队

- 收益本质相同（都是带出物资）
- 组队优势：**共享背包空间**，多一个人 = 多一份容量 = 每人多一份收益
- 设计目标：消除因物资背刺队友的动机
- 待解决问题：消极怠工（不干活只分物资）的防范机制

## 四、血条与生存系统

### 4.1 机械耐久度（血量条）

- 代表机器人躯体的物理完整性
- 被打光 → 机械躯体无法运作
- 显示方式：人型机械二维平面图，掉血=降水位（类似 CF 爆破模式人像血条）
- 考虑提供多种血条样式供玩家选择

### 4.2 信号强度（信号条）

- 玩家远程控制机器人的数据传输链路质量
- 被打光/被干扰至归零 → 玩家与载体断联 = 死亡
- 显示方式：滚动斜棱形长方框，绿白交替配色，降低时变黄→变红
- 信号干扰来源：时间推移（机制）、敌方黑客攻击（主动）

### 4.3 能量系统（技能蓝条）

- 使用技能消耗能量
- 能量不足可能导致：无法使用技能、部分功能受限
- 可通过道具补充
- 显示：屏幕中下，黄色（类似机油颜色）

### 4.4 伤害反馈

- **受击推算方向**：机器人躯体受力感应 → 反向推算弹药来向（非小地图显示）
- **被击中音效**：金属撞击声（金属弹头 vs 金属躯体）
- **命中反馈**：敌方被击中飚出金属碎屑（类比人类爆血雾）

## 五、机器人系统

### 5.1 三种机器人类型

**人形机器人：**
- 参考宇树机器人外形 + 核动力
- 全能型，无明显短板
- 可装载集成型载荷

**异形机器人：**
- 本质人型 + 外置模块
- 赛博朋克机械异肢风格
- 更多技能槽位

**关节分布式机器人：**
- 灵感：钢铁侠战甲
- 紧急时分离解体飞散逃离，之后再组装
- 可附身 AI 机器人实现敌方控制
- 专属技能：分离逃脱

### 5.2 强度与差异

- **无轻中重之分**，可互相转化
- 机器强度 = 使用什么部件（高性能部件=高耐久+低重量）
- 差异体现在：技能槽数量、背包容量、专属功能
- 取决于玩家的对局策略，而非固定职业

### 5.3 武器通用性

- 所有机器人通用所有武器（类似人类手持武器）
- 差异在于身体性能

### 5.4 外置模块（技能）

- 勾爪、爬墙、喷射弹跳等
- 玩家自选，受技能槽限制
- 可理解为"玩家自选技能"

### 5.5 外观

- 可修改表面图案（贴图、喷剂）
- 不能完全自定义形状
- 机器人皮肤：运营层面待定

### 5.6 维修

- 部件损坏 → 拆下维修或直接更换
- 随身携带备件 → 打架损坏快速替换恢复血量
- **维修套件**：整合各零部件，可现场生成所需部件
  - 本质是小型零件熔炼组件
  - 消耗对应材料
  - 可加入材料补充消耗
- 也可用电焊枪、热熔塑性机现场修复

## 六、枪械系统

### 6.1 定装火药武器

- 传统枪械原理
- 装填子弹 → 激发底火 → 发射弹头
- 子弹耗尽后更换弹夹
- 弹夹装载量系统

### 6.2 电磁弹射武器

- 充能进入激活状态
- 将弹夹内异形金属弹头加速发射
- 弹头类型效果：

| 弹头类型 | 效果 |
|---------|------|
| 金属射流弹 | 穿透性强 |
| 化学腐蚀弹 | 腐蚀性金属，持续伤害 |
| 强磁碎屑弹 | 命中碎裂附着机体，叠加电磁干扰，附着越多干扰越重 |

### 6.3 武器 UI 展示

- 右下角枪械 + 子弹数量（参考 Apex Legends）

## 七、算力与黑客系统

### 7.1 算力消耗行为

- **入侵破解**：类似三角洲"房卡"但更灵活
- 部分破解强相关特殊道具（无道具则无法破解）
- **激活场地设备**：短暂控制场景内强大道具
- **激活全图警戒系统**
- **侵入 AI 机器人**：制造迷惑行为、传递虚假信息、制造陷阱

### 7.2 信号受攻击反馈

- 画面噪点（数据回传受影响）
- 某些功能被禁用或效果降低
- 信息功能短时间内被 ban

### 7.3 防御黑客

- 本质：增加算力消灭攻击行为
- 多数黑客攻击效果 = 短时间干扰 + 算力消耗
- 反向入侵极困难，通常只能抵御

### 7.4 算力池共享

- 每个人自带算力设备决定局内算力规模
- 开局可选择将多少算力加入**共享池**
- 共享池算力：队友无需过问直接使用
- 无距离限制（同一数据传输链路）
- 极端情况：后期可直接**控制队友的载体**、使用他的装备
- 队友载体死亡后：观战视角 → 帮队友操作面板/破解系统（副驾驶模式）

### 7.5 算力 UI

- 屏幕中下位置（与能量条并列）
- 高亮光、微弱数字特效

## 九、动态地图建模

### 9.1 核心机制

- 玩家在特定区域内开启**扫描模式**并待一段时间
- 参考现实：录制场地图像 → 反向建模场景
- 特别区域有强屏蔽干扰 → 需完成任务/破解 → 直接复制建模信息 + 获取交互权限（门禁、特殊功能等）

### 9.2 建模效果

- 打开上帝视角 → 剥离贴图、只留关键线条的粗糙空间模型（类似 CS2 Dust2 白模）
- 可缩放、旋转
- 已建模区域**永久解锁**（类似空洞骑士买地图 + 探索点亮）
- 每解锁一个区域 = 建模空间中亮起一个区域

### 9.3 场景状态记录

- 玩家走过已建模区域 → 场景交互状态自动记录
- 再次经过时对比记录 → 有变化 = **有人来过** → 地图标红
- 队友分开走 → 队友路径自动为自己更新
- 发现队友路径场景变化 → 立即警戒

### 9.4 标记功能

- 在建模地图中标记地点/物资点
- 标记同步到第一人称视角 → 游戏内对应位置显示图标
- 容器标记 + 场景标记
- 与队友共享

### 9.5 设计参考

- 空洞骑士：购买地图 → 探索 → 点亮区域
- 猎杀对决：玩家推理"有人来过"
- 本系统：自动化+持久化+可分享

## 十二、团队协作

### 12.1 队伍规模

- 3人或4人（可能同时开放两种模式）

### 12.2 协作机制

- **共享背包空间**：多一个人=多容量=多收益，消除背刺动机
- **算力共享池**（见第七章）
- **队友载体控制**：队友死亡后可帮操作面板（见第七章）

### 12.3 撤离

- 不需要全队撤离
- 主力带关键物资走，剩下的人拖住敌人
- 收益事后分配

### 12.4 沟通系统

- 标记轮盘 + 快捷指令
- **自定义快速喊话**：预存储交互内容
- **语音风格自定义**：
  - 同一内容可切换语气（男声/女声/机器合成音）
  - 可附带 BGM 和音效（搞怪/调侃用）

### 12.5 待解决问题

- 消极怠工（不干活只分物资）的防范机制

## 十三、音频设计

### 13.1 材质音效

- 多材质地面：金属、泥土、碎石等
- 不同型号机器人走路声音可能不同
- 用于判断：什么类型的机器人在什么地面上、走了多远

### 13.2 信号干扰音效

- 数据传输受干扰时有杂音
- 但玩家终端可预处理数据，大幅隔绝大部分杂音影响
- 所以不会很严重，只是提示性存在

### 13.3 机械音

- 机器人自身运行机械音**不做传播效果**
- 理由：游戏中不只是玩家机器人，声音太杂；类似 CS2 切枪无声

### 13.4 战斗音效

- 金属撞击声（子弹命中机器人）
- 金属碎屑飙出音效
- 机器人移动时伺服/液压声（不给敌方听到）

## 十四、UI/HUD 架构

### 14.1 画面布局

| 位置 | 元素 | 样式 |
|------|------|------|
| 左上 | 机械耐久度 | 人型二维平面图，降水位式 |
| 左上 | 信号强度 | 滚动斜棱形长方框，绿白→黄→红 |
| 中下 | 能量条 | 黄色，机油质感 |
| 中下 | 算力条 | 高亮光+微弱数字特效 |
| 右下 | 武器+弹药 | 参考 Apex Legends |
| 右上 | 圆形小地图 | 已有雏形（锁定/跟随双模式） |
| 中心 | 十字准星 | 动态切换 |

### 14.2 其他 UI

- **撤离倒计时**：信号干扰增强的时间提示
- **队友状态**：血量、信号、位置
- **扫描进度条**：建模进度
- **受伤方向指示**：躯体受力感应推算（非小地图）
- **场景标记图标**：上帝视角标记→第一人称同步显示

### 14.3 可选功能

- 多种血条样式供玩家切换
- 自定义 UI 布局（待定）
