## 这是什么？

**你是一个坐在安全房间里远程操控机器人的操作者。** 你控制的机器人载体被派进全自动化矿场搜集物资。机器人被摧毁了，你的人身安全无恙——但带进去的装备永久丢失。

这不是又一款人类士兵射击游戏。这是一款**机器对机器的**游戏：

- **双重血条**：机械耐久度 + 信号强度（远程控制链路）。任一归零 = 死亡。
- **算力共享**：团队共享计算资源池，入侵破解、防御黑客、甚至操控队友的机器人。
- **动态地图建模**：扫描环境永久解锁线框3D地图。标记地点。场景变化标红→有人来过。
- **活的工厂生态系统**：地图本身是一条自动化产线——采矿机→运输车→熔炼站→加工中心→寄存站。产业链每一个环节都可以互动。

## 当前状态

**早期原型阶段。** 已有可玩测试场地（FPS移动+射击+敌人）。正在积极寻求合作者。

基于 [Kenney's Starter Kit FPS](https://github.com/KenneyNL/Starter-Kit-FPS)（CC0 资产）构建，新增：
- 动画 AK-74M 武器模型（PBR材质，8种关键帧动画）
- 双血条 HUD（机械耐久 + 信号强度）
- 能量条系统（信号随时间衰减）
- 小地图系统（锁定/跟随双模式）

## 快速开始

```bash
git clone https://github.com/Player-2048/RoboShooter.git
cd RoboShooter
# 用 Godot 4.6+ 打开项目，运行 scenes/main.tscn
```

**环境要求：** [Godot 4.6](https://godotengine.org/) 或更新版本。

## 为什么加入？

- **新颖的游戏设计** — 机器人搜打撤是一个未被开发的品类空间
- **纯 GDScript** — 易读、易改、易扩展
- **CC0 资产内置** — 下载就能跑、能测、能改
- **详细设计文档** — 14个系统的完整规格说明
- **新手友好代码** — 体量小、结构清晰、文档齐全
